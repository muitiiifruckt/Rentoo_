# Быстрый старт E2E тестов

## Шаг 1: Установка зависимостей

### Windows:
```bash
setup.bat
```

### Linux/Mac:
```bash
chmod +x setup.sh
./setup.sh
```

## Шаг 2: Убедитесь, что MongoDB запущена

Перед запуском тестов убедитесь, что MongoDB запущена и доступна на `localhost:27017`.

## Шаг 3: Запуск тестов

### Windows:
```bash
run_tests.bat
```

### Linux/Mac:
```bash
chmod +x run_tests.sh
./run_tests.sh
```

Или напрямую:
```bash
npm test
```

## Что тестируется

- ✅ Аутентификация (регистрация, вход, выход)
- ✅ Управление товарами (создание, просмотр, редактирование, удаление)
- ✅ Поиск и фильтрация товаров
- ✅ Бронирования (создание заявки, подтверждение, отмена)
- ✅ Профиль пользователя
- ✅ Базовые проверки работоспособности системы

## Режимы запуска

- `npm test` - все тесты в headless режиме
- `npm run test:ui` - с UI для отладки
- `npm run test:headed` - с видимым браузером
- `npm run test:debug` - режим отладки

## Просмотр отчета

После запуска тестов:
```bash
npm run test:report
```

## Устранение проблем

1. **Серверы не запускаются автоматически**: Убедитесь, что backend и frontend могут быть запущены вручную
2. **MongoDB недоступна**: Проверьте, что MongoDB запущена
3. **Тесты падают**: Проверьте логи в консоли и используйте `npm run test:ui` для отладки
